Industry 4.0 Research Team SFU


The Fourth Industrial Revolution consists of many new technologies that allow communication between physical and digital worlds. This means that communication between sensors, actuators, PLCs, computers, and many other components will be common and very important.


2021 Summer

Throughout the summer of 2021, I joined SFU's Industry 4.0 Research Team and approached this topic by gathering sensor data from an Arduino, and uploading this data to an OPC UA server via Raspberry Pi. A potentiometer and ultrasonic sensor were used to gather data onto the Arduino, and the Arduino was programmed to constantly upload the sensors' data to Serial. As the Raspberry Pi and Arduino were connected via USB, the Raspberry Pi could then read the data using the pyserial module in python. Then I set up an OPC UA server on the Raspberry Pi which uploaded the potentiometer and ultrasonic sensor values in realtime.

This data can now be accessed by any local OPC UA client. Initially, I was able to successfully connect to the OPC UA server via UA Expert, but found that Inductive Automation's Ignition software had much more useful features, such as scheduled reports. Using Ignition, I built a dashboard presenting realtime values of the sensors on graphs and gauges. I was also able to schedule daily reports which organizes the 24 Hour history of the sensors, and downloads it onto my PC as a PDF.


Future Plans

Future plans will involve working with actual PLCs and possibly Industrial machines. Communication between a real machine like a robot arm and a PLC would be a large step. Physical data could be gathered using sensors, then it could be communicated within the virtual environment and a physical response would be presented by the robots on the field.
